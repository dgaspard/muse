# Artifact Schemas v1.1

## Purpose
Defines the required and optional fields for all governance-derived artifacts in MUSE. Schemas are contracts for validation, ID generation, and audit.

---

## Epic
- id: string (EPIC-xxxx)
- governance: { source_document: { id, filename, checksum }, sections: [ { id, title, page_range } ] }
- title: string
- description: string
- lineage: { derived_from_governance: true }
- schema_version: v1.1

## Feature
- id: string (FEAT-xxxx)
- governance: { ... }
- title: string
- description: string
- lineage: { epic_id: EPIC-xxxx }
- schema_version: v1.1

## User Story
- id: string (STORY-xxxx)
- governance: { ... }
- title: string
- description: string
- acceptance_criteria: string[]
- lineage: { epic_id: EPIC-xxxx, feature_id: FEAT-xxxx }
- schema_version: v1.1

## Prompt
- id: string (PROMPT-xxxx)
- governance: { ... }
- prompt_purpose: string
- prompt_template: string
- lineage: { story_id: STORY-xxxx }
- schema_version: v1.1

---

## Rules
- All required fields must be present
- No extra fields affect ID generation
- Any change to this schema requires a new version and explicit approval
